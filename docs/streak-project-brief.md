# STREAK 案件概要書ドラフト

## 1. 案件名

STREAK

## 2. サービス概要

STREAK は、情報商材販売者向けの Web3 決済導入支援システムである。

加盟店はメールアドレスとパスワードで登録し、受取用ウォレットアドレスを設定する。商品を登録すると決済リンクが発行され、購入者はログイン不要で購入者情報を入力し、Polygon USDC で決済する。決済完了後、スマートコントラクトにより加盟店と STREAK へ手数料が自動配分される。

将来的にはクレジットカード、Apple Pay、Google Pay への対応を目指す。ただし Crossmint は審査不可のため不採用とし、代替プロバイダーを別途調査・選定する。

## 3. 提供価値

- クレジットカード決済への対応を目指す
- リザーブなしの即時決済
- 低手数料
- Web3 / USDC 決済への対応
- 情報商材販売者が決済導入しやすい仕組み
- 加盟店ごとに手数料率を変更できる柔軟性

## 4. ターゲット

初期ターゲットは日本国内の情報商材販売者とする。

想定利用者:

- オンライン教材販売者
- コンサルティング商品販売者
- デジタルコンテンツ販売者
- 高単価の情報提供サービス販売者

## 5. サービスの立て付け

STREAK は「決済代行」ではなく「Web3 決済インフラ」として設計・表現する。

STREAK が資金を預かるのではなく、購入者から加盟店ウォレットへ USDC を直接送金し、同時に STREAK 手数料をスマートコントラクトで自動配分する構成を基本とする。

## 7. 決済仕様

### 初期決済方法

- Polygon USDC のみ

### 将来対応予定

- クレジットカード
- Apple Pay
- Google Pay

### 手数料

STREAK 手数料は標準 2.5% とする。

ただし管理者は加盟店ごとに手数料率を変更できるものとする。

### 自動配分

標準設定では以下の配分とする。

- 加盟店: 97.5%
- STREAK: 2.5%

この配分はスマートコントラクトで自動実行する。

## 8. 認証仕様

### 加盟店ログイン

メールアドレスとパスワードによるログインとする。

低コスト運用を優先し、認証基盤は Supabase Auth を第一候補とする。

### メール認証

加盟店登録時、メールアドレス認証を必須とする。

メール認証後は、管理者の手動承認なしで自動的に利用開始できる。

### ウォレット

ウォレットはログイン用途ではなく、加盟店の受取先として登録する。

## 9. 加盟店登録項目

加盟店登録時に以下を入力させる。

- 店舗名
- 受取ウォレットアドレス
- メールアドレス
- パスワード
- 氏名 / 担当者名
- 電話番号
- 国
- 事業内容
- Web サイト URL
- 利用規約同意
- 反社会的勢力ではないことの確認

## 10. 購入者側仕様

購入者ログインは不要とする。

購入者は決済時に以下を必須入力する。

- 氏名
- メールアドレス
- 電話番号
- 郵便番号
- 住所
- 備考

決済完了後、購入者へ決済完了通知メールを送信する。

### 購入者メール記載内容

- 商品名
- 加盟店名
- 購入者名
- 金額 USDC
- 決済日時
- TX ハッシュ
- 問い合わせ先

## 11. 加盟店ダッシュボード

加盟店ダッシュボードには以下を用意する。

- 売上概要
- 商品管理
- 決済リンク一覧
- 決済履歴
- 注文履歴
- 設定
- 受取ウォレット設定
- 決済プロバイダー設定

売上表示は USDC を基本とする。

商品価格については、初期仕様では USDC 決済を前提とし、詳細な価格入力方式は実装前に確定する。

## 14. 法務・禁止商材方針

STREAK は情報商材販売者を対象にするため、利用規約では禁止商材・禁止行為を厳格に定義する。

### 禁止商材・禁止行為の例

- 違法商品・違法サービス
- 詐欺的または誤認を招く商品
- 確実な収益を保証する表現
- 「誰でも簡単に稼げる」などの誇大表現
- 投資助言・金融商品取引に関する無許可の助言
- 暗号資産・FX・株式等の利益獲得を目的とした助言商材
- 薬機法に抵触する表現を含む商品
- 医療・健康効果を不当にうたう商品
- アダルト関連商品
- ギャンブル関連商品
- マルチ商法・ねずみ講・紹介報酬を主目的とする商品
- 著作権・商標権等を侵害する商品
- 第三者の個人情報を不正に扱う商品
- 反社会的勢力に関係する取引
- その他 STREAK が不適切と判断する商品・サービス

### 特商法表示について

STREAK 自体は BtoB サービスとして開始するため、STREAK サイト上では特商法ページを初期表示しない方針。

ただし、加盟店が購入者に商品を販売する決済リンクについては、購入者が個人の場合に特定商取引法上の表示が必要となる可能性がある。実装前に、加盟店販売ページ側での表示要件を別途確認する。

## 16. 決済プロパイダー一次調査メモ

### Stripe

Stripe は候補になり得るが、STREAK の対象領域とは相性リスクが高い。

公式の制限リストでは、暗号資産関連は地域・用途により制限対象であり、また「短期間で簡単に稼げる」系のスキーム、誇大広告、投資・暗号資産で利益を得る方法の助言などは問題になり得る。さらに日本固有の禁止・制限項目として、投資商品や暗号資産で利益を得る方法に関する助言、オンラインゲームやギャンブル関連の予測・助言、占い、特商法表示がない事業者などが挙げられている。

そのため、情報商材販売者向けの決済基盤として Stripe を採用する場合は、審査落ち・アカウント停止・加盟店ごとの商材審査負荷が大きい可能性がある。

### Apple Pay / Google Pay

Apple Pay や Google Pay は単体の決済代行ではなく、カード決済プロバイダー側の対応可否に依存する。

Apple は国・地域ごとの対応を公開しており、Google Wallet も国・カード発行会社ごとの対応を案内している。したがって STREAK では、Apple Pay / Google Pay に対応した決済プロバイダーを選ぶ必要がある。

### 今後調査すべき候補

- Stripe
- Coinbase Commerce
- MoonPay
- Ramp Network
- Transak
- Banxa
- Wert
- 国内 PSP
- 暗号資産オンランプ事業者

比較観点:

- 日本法人または日本居住者で利用可能か
- 情報商材販売者を許容するか
- 暗号資産 / ステーブルコイン決済と相性がよいか
- クレジットカード対応
- Apple Pay 対応
- Google Pay 対応
- API / Checkout の導入難易度
- 審査条件
- 手数料
- 入金サイクル
- リザーブ有無
- 加盟店ごとのアカウント発行が必要か
- STREAK 側がプラットフォームとして集約できるか

## 17. デザイン・公開方針

- 言語は日本語のみ
- サービス表記は STREAK
- Vercel 仮 URL から開始
- 既存のロゴ・画像素材を使用
- 初期は過度な多言語対応をしない

## 18. 開発方針

既存コードを修正し続けるより、要件を整理した上で新規構築する方が生産性が高い。

理由:

- 既存コードに旧仕様と新仕様が混在している
- 文字化けや暫定実装が多い
- 認証・決済・管理機能が密結合している
- 決済プロバイダー前提が変わった
- お金を扱うため、設計から整えた方が安全
