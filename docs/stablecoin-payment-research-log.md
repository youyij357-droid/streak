# STREAK ステーブルコイン決済・法務調査ログ

最終更新: 2026-06-17  
対象: 日本国内向けの USDC 決済導入、決済プロバイダー選定、法務・運用リスク整理  
注意: これはプロダクト判断用の調査ログであり、法的助言ではない。実装・リリース前に弁護士、登録業者、決済/暗号資産コンプライアンス担当への確認が必要。

## 1. 現時点の結論

- 日本ではステーブルコインは主に資金決済法上の「電子決済手段」として整理される。電子決済手段等取引業者は金融庁・財務局への登録が必要。
- 金融庁の電子決済手段等取引業者登録一覧では、2025-03-04 現在、登録業者は SBI VCトレード株式会社 1社で、取扱電子決済手段は USDC。
- STREAK の MVP が「購入者が自分のウォレットから、加盟店ウォレットと STREAK 手数料ウォレットへ Polygon USDC を直接送る」構成なら、STREAK が購入代金を預からない/換金しない/カストディしない設計を維持することが重要。
- 一方で、STREAK が加盟店のために USDC を受領・保管・送金・売却・円転・返金代行する場合、電子決済手段等取引業、暗号資産交換業、資金移動業、収納代行/為替取引該当性、犯罪収益移転防止法上の本人確認などの論点が一気に強くなる。
- 日本向けに「USDC決済アグリゲーター」としてすぐ使える国内プロバイダーは限定的。現実的な候補は、(1) 非カストディのオンチェーン決済、(2) SBI VCトレード等の登録業者との提携、(3) Stripe/Circle/Coinbase等の海外・グローバルサービスの日本提供可否確認、の3系統。

## 2. 法律・規制メモ

### 電子決済手段としての USDC

- 金融庁の登録一覧では、SBI VCトレード株式会社が電子決済手段等取引業者として登録され、取扱電子決済手段が USDC とされている。
- 金融庁の同一覧は、取扱電子決済手段が資金決済法上の定義に該当することを確認したものだが、価値保証や推奨ではないという注意書きも付いている。
- SBI VCトレードのサイト上にも、電子決済手段は支払いを受ける者の同意がある場合に限り代価支払いに使える旨の注意書きがある。

### STREAK の設計で避けたい状態

- STREAK が購入者資金または USDC を一時的にでも受領・保有する。
- STREAK が加盟店に代わって回収、送金、換金、返金、残高管理を行う。
- STREAK が購入者/加盟店の秘密鍵・ウォレットを管理する。
- STREAK が「決済代行」「資金移動」「交換」「売買」「預かり」を連想させる表示や契約をする。

### MVPで維持したい説明

- STREAK は加盟店が作成した販売ページ/決済リンクを提供する Web3 決済インフラ。
- 購入者は自分のウォレットからスマートコントラクトへ送金する。
- スマートコントラクトは、加盟店ウォレットと STREAK 手数料ウォレットへ自動分配する。
- STREAK は購入代金を預からず、加盟店の売上残高を管理せず、法定通貨への交換を提供しない。

## 4. MVP判断

### 推奨する初期方針

- 決済方式: 非カストディのオンチェーン直接決済。
- 通貨: Polygon USDC のみ。
- 売上受領: 加盟店が登録した自己管理ウォレットへ直接着金。
- 手数料: スマートコントラクトで STREAK 手数料ウォレットへ自動分配。
- 換金: STREAKでは提供しない。必要に応じて加盟店自身が登録業者を使う。
- 返金: MVPではオンチェーン返金を自動代行しない。加盟店による個別対応、または後続機能で返金コントラクトを検討。

### 画面・規約に入れるべき表現

- 「STREAKは購入代金を預かりません」
- 「決済は購入者ウォレットからブロックチェーン上で実行されます」
- 「USDCは法定通貨ではなく、価格・為替・ネットワーク等のリスクがあります」
- 「加盟店は自己の責任で受取ウォレット、税務、商品提供、返金対応を管理します」
- 「禁止商材・不正利用・制裁対象・マネロン疑義がある場合は利用停止できる」

## 5. 要確認リスト

- 弁護士確認:
  - STREAKのスマートコントラクト自動分配が、資金移動業、電子決済手段等取引業、収納代行、前払式支払手段、媒介/代理に該当しない整理で足りるか。
  - STREAKの手数料受領が電子決済手段の取引業務に見えるリスク。
  - 加盟店審査、禁止商材、本人確認、KYT/AMLの最低ライン。
- SBI VCトレード確認:
  - 法人加盟店がUSDCを入出庫・売却・円転する導線。
  - API、法人アカウント、決済事業者との提携可否。
  - 対応チェーン。Polygon USDCの入出庫可否。
- Stripe確認:
  - 日本アカウントでCrypto/stablecoin payment methodが利用可能か。
  - USDCの対応チェーン、決済後の加盟店受取通貨、Connect対応、手数料。
  - 日本の電子決済手段規制との整理をStripe側がどう扱うか。
- Circle確認:
  - 日本法人でCircle Mint/APIが利用できるか。
  - Circleが日本国内で電子決済手段等取引業者として提供している範囲、または提携先利用前提か。
  - Polygon USDC、CCTP、ウォレットAPIの利用条件。
