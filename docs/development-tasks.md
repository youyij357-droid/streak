# STREAK 開発タスク分解

## 1. フェーズ0: 設計確定

- 要件定義書をレビューする。
- 画面一覧をレビューする。
- DB設計をレビューする。
- スマートコントラクト仕様をレビューする。
- MVPで扱う未確定事項を決める。
  - 商品価格は USDC 固定で始める。
  - 決済完了検知方式を決める。
  - サーバー署名方式を採用するか決める。
  - 法務レビュー範囲を決める。

## 2. フェーズ1: 新規実装の土台

- Next.js 16 のローカルドキュメントを確認する。
- 既存実装から使えるものと捨てるものを棚卸しする。
- 新規 App Router 構成を決める。
- 不要な旧仕様依存を外す。
- 環境変数を整理する。
- 共通レイアウト、認証導線、基本UIを用意する。
- Supabase クライアントを整理する。
- Resend クライアントを整理する。
- wagmi / viem の設定を整理する。

## 3. フェーズ2: DB / 認証

- Supabase マイグレーションを作成する。
- Enum を作成する。
- `profiles` を作成する。
- `merchants` を作成する。
- `merchant_settings` を作成する。
- `products` を作成する。
- `payment_links` を作成する。
- `orders` を作成する。
- `payments` を作成する。
- `email_events` を作成する。
- `audit_logs` を作成する。
- RLS ポリシーを設定する。
- 新規登録時のプロフィール作成処理を実装する。
- 管理者ロールの初期付与手順を用意する。

## 4. フェーズ3: 加盟店オンボーディング

- 加盟店登録画面を実装する。
- ログイン画面を実装する。
- メール認証待ち画面を実装する。
- 初期設定画面を実装する。
- 店舗情報保存処理を実装する。
- ウォレットアドレス検証を実装する。
- 停止中加盟店のアクセス制御を実装する。
- 管理者向け加盟店登録通知メールを実装する。

## 5. フェーズ4: 商品 / 決済リンク

- 商品一覧画面を実装する。
- 商品作成画面を実装する。
- 商品編集画面を実装する。
- 商品停止機能を実装する。
- 決済リンク自動発行を実装する。
- 決済リンク一覧画面を実装する。
- リンクコピーUIを実装する。
- 公開決済リンクの表示制御を実装する。

## 6. フェーズ5: スマートコントラクト

- Hardhat または Foundry の採用を決める。
- `StreakPaymentRouter` を実装する。
- USDC モックを用意する。
- 手数料配分テストを実装する。
- 二重決済防止を実装する。
- 緊急停止を実装する。
- サーバー署名検証を実装する。
- ローカルデプロイスクリプトを作成する。
- テストネットデプロイ手順を作成する。

## 7. フェーズ6: 購入者決済

- 決済リンク画面を実装する。
- 購入者情報フォームを実装する。
- 注文作成 API を実装する。
- 決済署名発行 API を実装する。
- ウォレット接続 UI を実装する。
- USDC approve を実装する。
- `pay` 実行を実装する。
- TX ハッシュ保存を実装する。
- 決済完了画面を実装する。
- 決済失敗 / 中断画面を実装する。

## 8. フェーズ7: 決済確定 / 通知

- TX 確認処理を実装する。
- 注文ステータス更新を実装する。
- `payments` 保存処理を実装する。
- 購入者向け決済完了メールを実装する。
- 加盟店向け決済通知メールを実装する。
- メール送信ログを保存する。
- メール再送の最低限の運用手順を用意する。

## 9. フェーズ8: 加盟店ダッシュボード

- 売上概要を実装する。
- 最近の注文を表示する。
- 注文履歴を実装する。
- 決済履歴を実装する。
- 設定画面を実装する。
- 受取ウォレット更新を実装する。

## 10. フェーズ9: 管理者画面

- 管理ダッシュボードを実装する。
- 加盟店一覧を実装する。
- 加盟店詳細を実装する。
- 加盟店停止 / 再開を実装する。
- 加盟店別手数料率変更を実装する。
- 注文一覧を実装する。
- 決済履歴を実装する。
- 手数料集計を実装する。
- 監査ログ保存を実装する。

## 11. フェーズ10: 公開準備

- Vercel 環境変数を設定する。
- Supabase 本番プロジェクトを用意する。
- Resend 本番設定を確認する。
- Polygon テストネットでE2E確認する。
- 禁止商材・利用規約まわりの表示を確認する。
- LP の文言を法務リスクが低い表現に調整する。
- 管理者アカウントの作成手順を固める。
- Vercel 仮 URL で公開する。

## 12. 最初の実装優先順位

1. DB マイグレーションと RLS
2. Supabase Auth の登録 / ログイン
3. 加盟店初期設定
4. 商品登録
5. 決済リンク表示
6. コントラクトのローカル決済
7. 注文作成から TX 保存までの一本線
8. 管理者の加盟店停止と手数料率変更
