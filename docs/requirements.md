# STREAK 要件定義書

## 1. 目的

STREAK は、情報商材・デジタルコンテンツ等を販売する加盟店が、低コストで Polygon USDC 決済リンクを発行できる Web3 決済インフラである。

MVP では、クレジットカード決済や Apple Pay / Google Pay は扱わず、購入者がログイン不要で USDC 決済できる最小構成を完成させる。

## 2. 基本方針

- STREAK は「決済代行」ではなく「Web3 決済インフラ」として表現する。
- STREAK は購入代金を預からない。
- 購入者からスマートコントラクトを経由して、加盟店ウォレットと STREAK 手数料ウォレットへ USDC を自動配分する。
- MVP の決済通貨は Polygon USDC のみとする。
- 加盟店認証はメールアドレス / パスワードを基本とし、Supabase Auth を第一候補とする。
- メール送信は Resend を第一候補とする。
- 法務・審査リスクを抑えるため、禁止商材・禁止行為を明確にし、管理者が加盟店を停止できるようにする。

## 3. 利用者

### 3.1 加盟店

情報商材、オンライン教材、コンサルティング商品、デジタルコンテンツ等を販売する事業者。

主な行動:

- アカウント登録
- メール認証
- 店舗情報登録
- 受取ウォレット設定
- 商品登録
- 決済リンク発行
- 注文・決済履歴確認

### 3.2 購入者

加盟店が発行した決済リンクから商品を購入するユーザー。購入者アカウントは作成しない。

主な行動:

- 決済リンクを開く
- 商品内容と金額を確認する
- 購入者情報を入力する
- ウォレット接続または送金操作を行う
- 決済完了メールを受け取る

### 3.3 管理者

STREAK 運営者。

主な行動:

- 加盟店の確認
- 加盟店の有効化 / 停止
- 手数料率の変更
- 注文・決済履歴の確認
- 不正取引の確認

## 5. 機能要件

### 5.1 加盟店登録

- 加盟店はメールアドレスとパスワードで登録できる。
- 登録時にメール認証を必須とする。
- メール認証完了後、管理者の手動承認なしで利用開始できる。
- 登録時または初回利用時に以下を入力する。
  - 店舗名
  - 受取ウォレットアドレス
  - 氏名 / 担当者名
  - 電話番号
  - 国
  - 事業内容
  - Web サイト URL
  - 利用規約同意
  - 反社会的勢力ではないことの確認

### 5.2 加盟店ログイン

- メールアドレス / パスワードでログインできる。
- ログイン後は加盟店ダッシュボードへ遷移する。
- 停止中の加盟店は主要機能を利用できない。

### 5.3 商品管理

- 加盟店は商品を登録できる。
- 商品には少なくとも以下を設定する。
  - 商品名
  - 商品説明
  - 価格 USDC
  - 公開状態
- 商品停止中は新規決済を受け付けない。

### 5.4 決済リンク

- 商品ごとに購入用 URL を発行する。
- 決済リンクは購入者ログインなしで表示できる。
- 加盟店または商品が停止中の場合は購入不可とする。

### 5.5 購入者情報入力

購入者は決済前に以下を入力する。

- 氏名
- メールアドレス
- 電話番号
- 郵便番号
- 住所
- 備考

### 5.6 USDC 決済

- 決済ネットワークは Polygon を前提とする。
- 決済トークンは USDC を前提とする。
- 購入者はウォレットでトランザクションを承認する。
- 決済完了後、TX ハッシュを保存する。
- オンチェーン結果と注文状態を照合できるようにする。

### 5.7 手数料配分

- 標準 STREAK 手数料率は 2.5% とする。
- 管理者は加盟店ごとに手数料率を変更できる。
- 決済時点の手数料率を注文に保存する。
- 配分はスマートコントラクトで実行する。

### 5.8 メール通知

- 購入者へ決済完了メールを送信する。
- 加盟店へ決済通知メールを送信する。
- 管理者へ加盟店登録通知を送信する。
- 送信失敗は決済成功を取り消さず、再送できるようログを残す。

### 5.9 管理者機能

- 管理者は加盟店一覧を確認できる。
- 管理者は加盟店を有効 / 停止にできる。
- 管理者は加盟店ごとの手数料率を変更できる。
- 管理者は注文・決済履歴・手数料合計を確認できる。

## 6. 非機能要件

### 6.1 コスト

- 初期構成は Vercel、Supabase、Resend、Polygon を中心に低コストで運用する。
- 重いバックエンド常駐サーバーは持たない。
- MVP ではジョブ基盤を最小化し、必要に応じて Vercel Cron または Supabase の機能を検討する。

### 6.2 セキュリティ

- 管理者 API は管理者権限を必須とする。
- 加盟店は自分の店舗・商品・注文のみ閲覧できる。
- 受取ウォレットアドレスは形式検証を行う。
- Webhook や決済確定処理は二重実行に耐える。
- 秘密鍵をアプリケーションサーバーに置かない構成を優先する。

### 6.3 監査性

- 注文ごとに金額、手数料率、配分額、TX ハッシュ、ステータスを保存する。
- 重要な管理操作は監査ログに残す。
- メール送信結果をログとして保存する。

### 6.4 法務・運用

- 利用規約で禁止商材・禁止行為を明示する。
- 加盟店は登録時に規約同意と反社確認を行う。
- STREAK サイト本体の特商法表示は MVP では必須表示しない方針とする。
- 加盟店の販売ページ側に必要な表示要件は実装前に別途確認する。

## 7. 未確定事項

- 商品価格入力を USDC 固定にするか、将来的な円表示を見越した項目を持つか。
- USDC コントラクトアドレスを環境ごとにどう管理するか。
- 本番公開前に法務レビューをどの粒度で行うか。
- オンチェーン決済完了の検知をフロント即時検知のみで始めるか、後段の照合ジョブをMVPに含めるか。
- カード決済プロバイダーの候補選定。
